System Roles

Two Interfaces, One System

🏭 Manufacturer / Seller

Dashboard Interface

Full control over the product catalog and real-time visibility into all incoming agent orders.

  • Manage product catalog (add, edit, archive products)
  • Set prices, units, minimum order quantities
  • View all orders from all agents in real-time
  • Filter orders by agent, product, date, status
  • Update order status (Confirmed, Shipped, Rejected)
  • Manage agent accounts (invite, suspend)
  • View analytics: revenue, top agents, top products
  • Export orders to CSV / PDF
  • Receive Telegram notification on new order
👤 Sales Agent

Mobile-first Order Interface

Streamlined interface optimized for on-the-go use — browse products, create orders at customer addresses.

  • Browse product catalog with photos and prices
  • Search and filter products by category
  • Add customer address / GPS location to order
  • Build cart and submit order with one tap
  • Add notes per product or per order
  • View own order history and statuses
  • Receive Telegram notifications on status changes
  • View daily / weekly personal sales stats
  • Multiple agents can work simultaneously

User Journey

End-to-End Order Flow

1

Agent Opens TWA via Telegram Bot

Agent taps the bot's menu button or inline link. Telegram passes initData with the user's Telegram ID. Backend validates HMAC signature and identifies the agent account.

2

Agent Browses Product Catalog

Products load from Supabase with photos, prices, and stock levels. Agent uses category filters and search to find items quickly. Quantities are adjusted with +/− buttons.

3

Agent Enters Customer Address

Agent types the delivery address or taps "Share GPS Location" to auto-fill coordinates. Telegram's LocationManager API is used for GPS. Notes can be added per item or for the whole order.

4

Agent Submits Order

Tapping the MainButton (Telegram's native green bottom button) confirms the order. Order is written to Supabase orders table with status pending. Supabase Realtime broadcasts the change.

5

Manufacturer Gets Notified Instantly

Supabase Realtime triggers a server-side function that sends a Telegram Bot notification to the manufacturer's chat: order summary, agent name, address, and total. Manufacturer's TWA dashboard updates live.

6

Manufacturer Reviews & Updates Status

Manufacturer opens the TWA, sees the order with full details, and taps Confirm / Reject / Ship. Status updates are written to Supabase and broadcast back to the agent's interface via Realtime.

7

Agent Gets Status Notification

Agent receives a Telegram push notification: "Your order #ORD-041 has been confirmed ✅". Agent can tap it to open the TWA and see full details in their order history.

Supabase SQL (Migration)

SQL
-- Enable UUID extension
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- USERS
CREATE TABLE users (
  id              uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  telegram_id     bigint UNIQUE NOT NULL,
  telegram_username text,
  full_name       text NOT NULL,
  phone           text,
  role            text NOT NULL CHECK (role IN ('manufacturer', 'agent')),
  is_active       boolean DEFAULT true,
  manufacturer_id uuid REFERENCES users(id),
  created_at      timestamptz DEFAULT now()
);

-- PRODUCTS
CREATE TABLE products (
  id              uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  manufacturer_id uuid NOT NULL REFERENCES users(id),
  name            text NOT NULL,
  description     text,
  category        text,
  unit            text,
  price           numeric(12,2) NOT NULL,
  min_order_qty   integer DEFAULT 1,
  stock_qty       integer,
  image_url       text,
  is_active       boolean DEFAULT true,
  created_at      timestamptz DEFAULT now(),
  updated_at      timestamptz DEFAULT now()
);

-- ORDERS
CREATE TABLE orders (
  id               uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  order_number     text UNIQUE NOT NULL,
  agent_id         uuid NOT NULL REFERENCES users(id),
  manufacturer_id  uuid NOT NULL REFERENCES users(id),
  status           text NOT NULL DEFAULT 'pending'
                   CHECK (status IN (
                     'pending','confirmed','shipped',
                     'delivered','rejected','cancelled')),
  delivery_address text,
  delivery_lat     numeric(10,7),
  delivery_lng     numeric(10,7),
  customer_name    text,
  customer_phone   text,
  notes            text,
  total_amount     numeric(12,2) DEFAULT 0,
  rejected_reason  text,
  created_at       timestamptz DEFAULT now(),
  updated_at       timestamptz DEFAULT now()
);

-- ORDER ITEMS
CREATE TABLE order_items (
  id           uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  order_id     uuid NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
  product_id   uuid REFERENCES products(id),
  product_name text NOT NULL,
  unit_price   numeric(12,2) NOT NULL,
  quantity     integer NOT NULL CHECK (quantity > 0),
  subtotal     numeric(12,2) GENERATED ALWAYS AS (unit_price * quantity) STORED,
  item_notes   text
);

-- NOTIFICATIONS LOG
CREATE TABLE notifications (
  id           uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  recipient_id uuid NOT NULL REFERENCES users(id),
  order_id     uuid REFERENCES orders(id),
  message      text,
  sent_at      timestamptz DEFAULT now(),
  status       text DEFAULT 'sent'
);

-- Enable Realtime on orders
ALTER PUBLICATION supabase_realtime ADD TABLE orders;
ALTER PUBLICATION supabase_realtime ADD TABLE order_items;

-- Auto-update updated_at
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER orders_updated_at
  BEFORE UPDATE ON orders
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();

-- Auto-generate order number
CREATE SEQUENCE order_seq START WITH 1;
CREATE OR REPLACE FUNCTION gen_order_number()
RETURNS TRIGGER AS $$
BEGIN
  NEW.order_number = 'ORD-' || LPAD(nextval('order_seq')::text, 5, '0');
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER before_insert_order
  BEFORE INSERT ON orders
  FOR EACH ROW EXECUTE FUNCTION gen_order_number();

-- RLS Policies
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE products ENABLE ROW LEVEL SECURITY;
ALTER TABLE orders ENABLE ROW LEVEL SECURITY;
ALTER TABLE order_items ENABLE ROW LEVEL SECURITY;

-- Agents: see only their own orders
CREATE POLICY agent_orders ON orders
  FOR ALL USING (agent_id = auth.uid() OR manufacturer_id = auth.uid());
